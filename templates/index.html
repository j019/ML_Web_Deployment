II.

model.py — Training the Model

This script reads the housing dataset, selects six columns as features, fits a Ridge regression against price, and freezes the trained model into model.pkl so app.py never has to retrain it.

model.pypython
# Importing the libraries
import numpy as np
import pandas as pd
import pickle

dataset = pd.read_csv('house_price_prediction.csv')

top_6 = ['sqft_living15','sqft_above','grade',
'condition','bedrooms','floors']

X = dataset.loc[:, top_6]
y = dataset['price']

from sklearn.linear_model import Ridge
regressor = Ridge(random_state=7)

# Fitting model with training data
regressor.fit(X, y)

# Saving model to disk
pickle.dump(regressor, open('model.pkl','wb'))

# Loading model to compare the results
model = pickle.load(open('model.pkl','rb'))
print(model.predict([[1440,1730,7,3,3,2]]))
top_6

Feature order is the contract. The list ['sqft_living15','sqft_above','grade','condition','bedrooms','floors'] defines the exact column order Ridge learns weights against — this order has to be respected everywhere downstream.

Ridge()

Why Ridge, not plain linear regression. Ridge adds L2 regularization, which keeps coefficients steadier when features like sqft_living15 and sqft_above are correlated with each other.

pickle.dump

model.pkl is the handoff. This file is the only thing app.py needs from this script — a frozen snapshot of the fitted regressor, ready to run .predict() with no dataset in sight.

III.

app.py — Serving Appraisals

Flask loads the frozen model once at startup, renders this entry form on /, and on /predict turns the submitted fields into a figure.

app.pypython
import numpy as np
from flask import Flask, request, jsonify, render_template
import pickle

app = Flask(__name__)
model = pickle.load(open('model.pkl', 'rb'))

@app.route('/') # index or landing page of website
def home():
    return render_template('index.html')

# 127.0.0.1:9000/predict
@app.route('/predict',methods=['POST']) # post method is used to send parameters in http request
def predict():
    '''For rendering results on HTML GUI'''
    int_features = [int(x) for x in request.form.values()]
    final_features = [np.array(int_features)]
    prediction = model.predict(final_features)
    output = round(prediction[0], 2)
    return render_template('index.html',
        prediction_text='Predicted House price is {}'.format(output))

if __name__ == "__main__":
    app.run(host="0.0.0.0",port=9000) # EC2 on AWS

Loaded once, reused forever. The pickle load happens at import time, outside any route — deserialized a single time when the server starts, not on every request.

request.form.values()

Order-dependent by design. This pulls values in the order the fields appear on this page, not by name. It only works because this form's field order matches top_6 exactly.

int(x)

Everything is cast to int. Fine for whole numbers like grade or bedrooms, but throws a ValueError on a decimal — worth knowing before this goes live.

Critical — Field Order Must Match

The trained model expects features in this exact order: sqft_living15 → sqft_above → grade → condition → bedrooms → floors.

Since request.form.values() carries no field names, the inputs above must appear in this same order, top to bottom, or the model will silently value the wrong columns. The form on this page currently matches correctly.

IV.

Bringing the Register Online

01

Set up the environment

Create an isolated environment and install dependencies before touching the model.

$ python -m venv venv && source venv/bin/activate
$ pip install flask numpy pandas scikit-learn
02

Train once, freeze the artifact

Run model.py locally to generate model.pkl. This only needs to happen once, or whenever the dataset or features change — the server never retrains.

$ python model.py
03

Confirm the form matches the model

Double-check this page's input order against top_6 in model.py. This is the single most common source of silently wrong appraisals.

04

Run it locally first

Sanity-check on your own machine before touching a server.

$ python app.py
# visit http://127.0.0.1:9000
05

Move it to an EC2 instance

Since app.run(host="0.0.0.0", port=9000) is already set, the server will accept external traffic. Open the port in your security group, then start it the same way.

$ scp -r ./project ec2-user@<your-ec2-ip>:~/app
$ ssh ec2-user@<your-ec2-ip>
$ cd ~/app && pip install -r requirements.txt
$ python app.py
06

Harden it for production

Flask's built-in server is fine for testing, not for real traffic. Swap it for Gunicorn behind Nginx, and keep it alive with a process manager.

$ pip install gunicorn
$ gunicorn --bind 0.0.0.0:9000 app:app
V.

Before This Entry Opens to the Public

  • model.pkl exists and was generated from the same feature set the form submits.
  • requirements.txt pins flask, numpy, pandas, and scikit-learn versions so the pickle loads the same way on the server as it did locally.
  • Security group / firewall allows inbound traffic on port 9000, or whichever port you choose.
  • Form field order on this page matches top_6 exactly — sqft_living15, sqft_above, grade, condition, bedrooms, floors.
  • Debug mode is off before this is public — app.run() currently has no debug=True, which is correct for production.
